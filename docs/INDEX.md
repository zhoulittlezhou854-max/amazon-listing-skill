## 使用指南

### 新文档创建
1. **确定文档类型**：PRD、知识库、进度、审核、总结
2. **选择对应目录**：将文件放入正确的子目录
3. **命名规范**：使用描述性名称，如 `feature-specification.md`
4. **更新索引**：更新本文件和子目录索引

### 文档维护
- 定期审查和更新文档
- 删除过时文档到 `archive/deprecated_docs/`
- 保持目录结构清晰
