## 目的
存放项目所有文档文件，按类型分类组织。

## 内容结构
```
docs/
├── INDEX.md (此文件)
├── prd/                 # 产品需求文档
├── knowledge-base/      # 知识库和技术文档
└── input_tables_spec.md # 输入表规范
```

## 文件说明
| 目录 | 说明 | 文件数量 |
|------|------|----------|

## 使用指南

### 新文档创建
1. **确定文档类型**：PRD、知识库、输入规范
2. **选择对应目录**：将文件放入正确的子目录
3. **命名规范**：使用描述性名称，如 `feature-specification.md`
4. **更新索引**：更新本文件和子目录索引

### 文档维护
- 定期审查和更新文档
- 过程性历史文档迁入 `archive/deprecated_docs/`
- 保持线上运行所需文档精简可读
