# processed/ INDEX

## 目的
存放处理后的数据文件，是原始数据经过清洗、转换后的结果。

## 内容结构
```
processed/
├── INDEX.md (此文件)
└── (空目录)
```

## 数据特性
- 可重新生成：数据可以从 raw/ 数据重新处理得到
- 中间结果：可能包含多个处理阶段的结果
- 格式统一：数据格式应保持一致，便于下游使用

## 使用指南
1. 处理脚本应输出到此目录
2. 文件命名应反映处理阶段
3. 大文件应使用 .gitignore 排除
4. 定期清理不再需要的中间文件
